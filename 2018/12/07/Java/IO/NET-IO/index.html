网络IO

阻塞和非阻塞

阻塞和非阻塞是进程在访问数据的时候，数据内是否准备就绪的一种处理方式，当数据没有准备的时候，

阻塞往往需要等待缓冲区中的数据准备过后才能处理其他的事情，否则一直等待在那里。
非阻塞则是当我们的进程访问我们的数据缓冲区的时候，数据没有准备好的时候，直接返回，不需要等待，数据有的时候，也直接返回。

同步和异步的方式

同步和异步都是基于应用程序和操作系统处理IO时间锁采用的方式，

  • 同步应用程序要直接参与IO读写的操作，异步则是所有的IO读写交给操作系统去处理
  • 同步的方式在处理IO事件的时候，必须阻塞在某个方法上面，等待我们的IO事件完成（阻塞IO事件或者通过轮询IO事件的方式），对于异步而言，所有的IO读写都交给了操作系统，这个时候，可以去做其他的事情，并不需要去完成真正的IO操作，当操作IO完成后，给应用一个通知就可以。

阻塞到IO事件

阻塞到read或者write，这个时候就不能做自己的事情。因此采用这种方式，对线程的性能开销比较大。

IO事件的轮询

或者也称为多路复用技术(select模式)，读写事件交给一个单独的线程来处理，完成IO事件的注册，轮询读写缓冲区（查看数据是否准备好）以及通知相应的读写线程等。这样就不会阻塞主线程，使得主线程可以继续做其他的事情。这个时候阻塞的不是所有的IO线程，而是select这个线程。

Java NIO

JDK版本 技术点
JDK1.4之前 引入了BIO
JDK1.4 引入了多路复用技术(select)，同步非阻塞
JDK1.7 epoll，异步非阻塞方式，AIO

对于网络通信而言，NIO和AIO并没有改变网络通信的基本步骤，只是在原来的基础上做了些改进。

对于三次握手的方式建立的连接性能开销比较大，因此通过减少连接的次数，来提升性能。也就是对读写管道进行一个抽象(Channel)。Channel是在一个TCP连接之间抽象，一个TCP连接可以对应多个管道。
