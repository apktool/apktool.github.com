HBase之RegionServer启动流程

Posted on 2020-09-26 In HBase源码

HBase RegionServer的启动流程代码大约4K行，看起来非常繁琐，但实际上并不算复杂。主要完成了参数初始化，连接HMaster和Zookeeper，创建线程池等。

入口

这部分主要完成了获取hbase-site.xml和hbase-defalut.xml中的配置信息（存储倒Configuration），接着将该配置信息作为参数，通过反射的方式创建HRegionServer实例。最后调用HRegionServer中的run()方法，启动HRegionServer。

HRegionServer的构造方法

构造方法作为HRegionerver实例化时的重要一步，它完成了大量的初始化工作。简单来说就是初始化和校验工作。其中初始化包括参数初始化，线程池初始化，RPC Client初始化，以及其他初始化。

HRegionServer的run方法

整个run方法实际上就由两个 try ... catch 语句和若干if组成。这两个 try ... catch 语句描述了RegionServer正常工作时的样子。后续的if语句描述了RegionServer关闭后的一些步骤。

preRegistrationInitialization

initializeZooKeeper

  1. 检查ZNode/hbase/master是否已经被赋值，如果没有，则阻塞等待;
  2. 检查ZNode/hbase/running是否已经被赋值，如果没有，则阻塞等待;
  3. 初始化rspm(region server procedure manager)

setupClusterConnection

一般在使用HBase API时，会创建连接，默认情况下会使用 ConnectionImplementation 进行实例化。而在HRegionServer在建立连接的时候，启用短路的方式，则会使用ShortCircuitingClusterConnection进行实例化连接。从类的继承关系来看，ShortCircuitingClusterConnection继承自ConnectionImplementation。

而ShortCircuitingClusterConnection的实现逻辑也非常简单。如果从配置文件中获取到的当前servername与传入的serername一致，那么取本地的链接便可。

rpcClient

最后初始化rpcClient，默认初始化为NettyRpcClient。

try … catch …

未停止并且未终止

reportForDuty是个非常重要的方法，完成了下述，但不止下述的功能。

  1. 重置 requestCount rpcGetRequestCount rpcScanRequestCount rpcMultiRequestCount rpcMutateRequestCount
  2. 创建regionserver status

handleReportForDutyResponse()是个非常重要的方法，完成了下述，但不止下述的功能。

  1. createMyEphemeralNode()创建zookeeper上的所有临时节点
  2. initializeFileSystem()初始化HDFS上面的目录结构
  3. setupWALAndReplication()初始化WAL和replica
  4. startServices()启动服务，包括flusher, compactsplit，leases，movedRegionsCleaner，storefileRefresher，walRoller，procedureResultReporter，executorService，......
  5. startReplicationService()启动并初始化repilca相关的服务
